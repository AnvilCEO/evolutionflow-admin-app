# E2E 테스트 시나리오: 신청 -> 관리자 승인 -> My Page 반영

## 목적
- 사용자 신청(강사/워크샵/트립&이벤트)이 My Page와 관리자 신청관리 화면에서 일관되게 반영되는지 검증한다.
- 관리자 승인 후 사용자 상태(신청 상태, 등급)가 기대대로 변경되는지 검증한다.

## 범위 및 매핑
- 강사 신청: `requests/teacher`
- 워크샵 신청: `requests/workshop`
- 트립&이벤트 신청: `requests/schedule`
- 관리자 화면: `관리자 > 신청관리 > 신청내역` (`/admin/requests`)
- 사용자 화면: `My Page > 등록내역` (사용자 서비스 앱)

## 공통 전제조건
1. 테스트 계정 2개 준비
2. 사용자 계정: 초기 등급 `MEMBER` (강사 시나리오 검증용)
3. 관리자 계정: 권한 `ADMIN`
4. 테스트 데이터 식별자(prefix): `E2E-{yyyymmddhhmm}`
5. 사용자 계정에 동일 타입의 기존 `pending` 신청이 없도록 정리
6. 승인 반영 지연을 고려해 상태 확인 시 최대 30초까지 polling

## 공통 검증 포인트
1. 사용자 신청 직후 My Page 등록내역 상태가 `신청중(pending)`으로 노출된다.
2. 관리자 신청관리 목록에서 동일 신청 건이 `심사중(pending)`으로 조회된다.
3. 관리자 승인 후 관리자 목록 상태가 `승인(approved)`으로 변경된다.
4. 사용자 My Page 등록내역 상태가 `승인(approved)`으로 변경된다.
5. 강사 신청 승인 시 사용자 등급이 `INSTRUCTOR`로 변경된다.

## TC-01 강사 신청 승인 플로우
### 테스트 데이터
- 이름: `E2E-Teacher-{timestamp}`
- 이메일/전화/SNS/소개: 식별 가능한 임시값

### 단계
1. 사용자 로그인 후 `강사 신청하기`를 작성/제출한다.
2. 사용자 `My Page > 등록내역 > 강사 신청`으로 이동한다.
3. 상태가 `신청중`인지 확인한다.
4. 관리자 로그인 후 `신청관리 > 강사 신청` 탭으로 이동한다.
5. 해당 신청 건을 검색/확인한다.
6. 신청 건의 상태가 `심사중`인지 확인 후 `승인`을 클릭한다.
7. 사용자 세션으로 돌아가 `My Page > 강사 신청`을 새로고침한다.
8. 상태가 `승인`으로 변경되었는지 확인한다.
9. `My Page > 프로필(또는 계정정보)`에서 등급이 `INSTRUCTOR`로 변경되었는지 확인한다.

### 기대 결과
- 신청 상태: `pending -> approved`
- 사용자 등급: `MEMBER -> INSTRUCTOR`
- 관리자 목록과 사용자 My Page의 최종 상태가 일치한다.

## TC-02 워크샵 신청 승인 플로우
### 테스트 데이터
- 워크샵명: `E2E-Workshop-{timestamp}`
- 강사명/일시/가격/정원/소개: 유효한 임시값

### 단계
1. 사용자 로그인 후 `워크샵 신청`을 작성/제출한다.
2. 사용자 `My Page > 등록내역 > 워크샵 신청`에서 상태 `신청중`을 확인한다.
3. 관리자 로그인 후 `신청관리 > 워크샵 신청` 탭으로 이동한다.
4. 해당 신청 건을 확인하고 `승인` 처리한다.
5. 사용자 `My Page > 워크샵 신청`을 새로고침한다.
6. 상태가 `승인`으로 변경되었는지 확인한다.

### 기대 결과
- 신청 상태: `pending -> approved`
- 신청 정보(제목/일시/신청일)가 관리자/사용자 화면에서 동일하게 보인다.

## TC-03 트립&이벤트 신청 승인 플로우
### 테스트 데이터
- 클래스명: `E2E-TripEvent-{timestamp}`
- 타입/일시/시간/장소/정원/설명: 유효한 임시값

### 단계
1. 사용자 로그인 후 `트립&이벤트 신청`을 작성/제출한다.
2. 사용자 `My Page > 등록내역 > 트립&이벤트 신청`에서 상태 `신청중`을 확인한다.
3. 관리자 로그인 후 `신청관리 > 스케줄 신청` 탭으로 이동한다.
4. 해당 신청 건을 확인하고 `승인` 처리한다.
5. 사용자 `My Page > 트립&이벤트 신청`을 새로고침한다.
6. 상태가 `승인`으로 변경되었는지 확인한다.

### 기대 결과
- 신청 상태: `pending -> approved`
- 등록내역 반영이 관리자/사용자 화면에서 일관된다.

## 권장 자동화 구현 기준 (Playwright)
1. 브라우저 컨텍스트 2개를 분리한다.
2. 컨텍스트 A: 사용자 앱(신청, My Page 검증)
3. 컨텍스트 B: 관리자 앱(신청관리 승인)
4. 테스트 데이터는 각 케이스 실행 시 동적으로 생성한다.
5. 승인 후 My Page 반영은 polling 방식으로 최대 30초 확인한다.
6. 실패 시 스크린샷, 네트워크 응답, 신청 ID를 아티팩트로 저장한다.
